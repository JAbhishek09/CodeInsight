## 4. Authentication Flow

```
Register/Login → POST /api/auth/register|login
  → bcryptjs hash/compare
  → JWT signed with JWT_SECRET (30-day expiry — see issue #1)
  → Token stored in localStorage ('codeinsight_token')

Every API request:
  → axios interceptor reads localStorage → Authorization: Bearer <token>
  → protect middleware: jwt.verify → req.user = User.findById(decoded.id)
  → 401 response → axios interceptor clears storage + redirects to /login

Token refresh: NONE — JWT is 30 days, no refresh token mechanism
```

## 7. Architectural Concerns (Summary — detailed in other reports)

1. **JWT: 30-day expiry, no refresh token** — session cannot be invalidated server-side
2. **sync.controller N+1 problem** — one DB write per submission (can be 100+ writes)
3. **In-memory nonce cache** — lost on server restart; allows replay attacks during restart window
4. **dbService.js** — large in-memory fallback layer that is never actually used by controllers (controllers use Mongoose directly). Creates a misleading code path and a hardcoded demo password.
5. **user.controller.js and user.routes.js are empty** — dead files
6. **`Spinner..tsx`** — double-dot filename typo (never imported anywhere)
7. **Missing `@google/generative-ai`** in backend package.json dependencies
8. **sync.controller** doesn't prevent duplicate submissions — $push always appends
9. **CORS is wide open** — `app.use(cors())` with no origin restriction
10. **No rate limiting on sync endpoint** — one user can trigger unlimited Gemini/LC/CF calls
11. **generateToken uses hardcoded 30d** — ignores JWT_EXPIRES_IN env var
12. **DashboardPage calls `refreshUser()` inside useEffect with no deps** — potential infinite loop in StrictMode
13. **analyzeCode instantiates GoogleGenerativeAI on every call** — should be singleton
14. **Missing `recharts` dependency** — chart components (`TopicBarChhart.tsx`, `WeeklyLineChart.tsx`, `DifficultyPieChart.tsx`) import from recharts but it's not in package.json
